2.2. Hyperlink¶

Paragraphs contain text and may contain inline markup.

  • emphasis： *emphasis*

  • strong emphasis： **strong emphasis**

  • interpreted text： `interpreted text`

  • inline literals： ``inline literals``

  • standalone hyperlinks： http://www.python.org

  • external hyperlinks： Python

    demoe:

    click me link to Python

    Python_
.. _Python: http://www.python.org

    click me link to google

    `google <https://www.google.com.tw/>`_

    click me go to

    .. _linking-Sphinx.Basic:
:ref:`linking-Sphinx.Basic`

:ref:`go to <linking-Sphinx.Basic>`

  • internal cross-references： go_to_example

    The “_example” target above points to this paragraph.

    syntax:

    example_

.. _example:

The "_example" target above points to this paragraph.

  • footnote references: 1 ([1]_)

  • citation references: [CIT2002] ([CIT2002]_)

  • substitution references: Hello World!

    syntax:

    |replace demo|

.. |replace demo| replace:: Hello World!

  • replace and link: mathjax demo

    syntax:

    |mathjax|_

.. |mathjax| replace:: mathjax demo
.. _mathjax: https://www.mathjax.org/#samples

  • function: py:function:: def start(file):

    def start(file):

2.4. List¶

2.4.1. Bullet lists:¶

  • This is a bullet list.

  • Bullets can be “*”, “+”, or “-“.

2.4.2. Enumerated lists:¶

  1. This is an enumerated list.

  2. Enumerators may be arabic numbers, letters, or roman numerals.

2.4.3. Definition lists:¶

what

Definition lists associate a term with a definition.

how

The term is a one-line phrase, and the definition is one or more paragraphs or body elements, indented relative to the term.

2.4.4. Field lists:¶

what

Field lists map field names to field bodies, like database records. They are often part of an extension syntax.

how

The field marker is a colon, the field name, and a colon.

The field body may contain one or more body elements, indented relative to the field marker.

2.4.5. Option lists¶

for listing command-line options:

-a

-b file

options can have arguments and long descriptions

--long

options can be long also

--input=file

long options can also have arguments

/V

DOS/VMS-style options too
